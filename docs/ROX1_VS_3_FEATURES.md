# 1.0 与 3.0 功能对比与补齐建议

## 当前状态

- **默认 UI**：已是 1.0 外壳（导航、模式切换、侧栏、多视图）。
- **数据**：K 线/指标已接 Ashare 备用（东方财富失败时用腾讯/新浪），数据能加载出来优先保证。

## 1.0 有、3.0 当前没有或较简的功能

| 1.0 功能 | 3.0 现状 | 建议 |
|----------|----------|------|
| **宏观仪表盘** | 有 `/api/market/macro`，经典界面里是占位 | 在经典界面「宏观」视图里接 3.0 的 macro 接口，做成 1.0 那种多卡片（国债、M2、社融、CPI、汇率、PMI、Shibor 等） |
| **每周推荐** | 有 `/api/market/weekly`，经典界面里是占位 | 在「每周推荐」视图里接 3.0 的 weekly 接口，展示 1.0 风格的策略说明 + 334 仓位法则 + 股票卡片 |
| **个股诊断（完整表单）** | 3.0 有分析接口，但经典界面侧栏没有 1.0 那种「现价/止损/目标/获利盘/量能/共振等级/模拟建仓/历史记录」 | 在经典界面侧栏「个股诊断」里恢复 1.0 的表单和沙盘推演入口，对接到 3.0 的 `/api/analysis/*`、`/api/stock/*` |
| **知识库** | 有 `/api/kb/search`，经典界面里是占位 | 在「知识库」视图里接 3.0 的 kb 接口，做成 1.0 的搜索 + 本地/联网/混合 + 结果列表 |
| **交易系统** | 有 `/api/trade/*`，经典界面里是占位 | 在「交易」视图里接 3.0 的账户/下单/历史，做成 1.0 的模拟盘/实盘切换、同步实盘、新建交易表单 |
| **社区跟单** | 1.0 有社区流 | 3.0 有 `/api/market/community/feed` 可接；若优先级低可先保留占位 |
| **系统状态** | 1.0 有系统状态页 | 3.0 有 `/api/system/system/status` 等，可做一页「系统状态」 |

## 建议的加入顺序

1. **先做「市场看板」数据稳定**（已完成：Ashare 备用）。
2. **宏观仪表盘**：接口已有，只差在经典界面里把「宏观」视图做成 1.0 风格多卡片并接 `/api/market/macro`。
3. **每周推荐**：接口已有，在「每周推荐」视图里接 `/api/market/weekly`，恢复 1.0 的 334 法则 + 策略展望 + 股票卡片。
4. **个股诊断**：在经典界面侧栏恢复 1.0 的完整诊断表单和沙盘推演，对接到 3.0 的分析与诊断接口。
5. **知识库 / 交易 / 社区 / 系统状态**：按需逐步接 3.0 已有 API，把占位换成真实内容。

如果你愿意，可以从「宏观」和「每周推荐」两个视图先做，再做「个股诊断」侧栏与「知识库/交易」。
