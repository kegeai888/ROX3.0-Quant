# 通达信截图 → Rox Quant 可引用要点

根据你提供的通达信金融终端截图，整理可直接借鉴到我们软件的功能、布局与数据项，便于后续迭代对齐。

---

## 一、市场看板（对应 `view-market`）

### 1. 底部指数条（强烈建议引用）
- **形式**：底部一条横条，多指数并排。
- **内容**：上证指数、深证成指、科创综指、北证50、创业板指、科创50。
- **每项**：现价、涨跌、涨跌幅%、成交额(亿)，红涨绿跌。
- **Rox 现状**：目前只有上证头部一块；可增加「底部指数概览」组件，数据可用现有/扩展指数接口。

### 2. 多指数分时并排（可选）
- **形式**：上证、深证、沪深当月 三块并列，每块独立：指数头 + 分时/五日/1分…/日线/周线/月线。
- **Rox 现状**：当前仅上证一块；若要做「多维看盘」，可把单指数块做成可复制多块、可切换指数。

### 3. 板块列表 + 成分股（已有，可加强）
- **形式**：Tab「所有板块 / 行业板块 / 概念板块 / 风格板块 / 地区板块」→ 表格：板块名称、涨幅%、涨速% → 点击展开成分股列表（序号、代码、名称、现价、涨幅%、涨跌、涨速%）。
- **Rox 现状**：板块热力、板块资金流向已有；可增加「点击板块展开成分股」和 Tab 切换行业/概念。

### 4. 资讯双栏（可选，放市场或独立「资讯」）
- **形式**：左侧「新闻研报」（日期、类型、公司、标题），右侧「最新公告」（日期、类型、摘要）；底部仍是指数条。
- **Rox 现状**：可作市场看板子区域或独立「资讯」视图，需公告/新闻 API。

## 二、个股诊断（对应 `view-diagnosis`）

### 1. 左侧分类导航（建议引用）
- **列表**：最新提示、资金动向、公司资料、股本结构、股东研究、分红融资、财务分析、经营分析、资本运作、研报评级、热点题材、公司资讯、主力持仓、行业分析、深度分析。
- **作用**：点击一项，主区只显示该分类内容，板块清晰、不串位。
- **Rox 现状**：诊断目前是「一段式」结果；可改为「左侧分类 + 右侧内容」，先实现若干项（如最新提示、资金动向、公司资料、财务分析），其余用「敬请期待」或链到 3.0。

### 2. 公司概要 / 最新提示（建议引用）
- **公司层面**：公司地位、主营业务、概念题材（标签+详情）、可比公司（国内/境外）、市场人气排名、行业人气排名、研究行业。
- **主要指标表**：市盈率(TTM/LYR)、营业收入及同比、归母净利润及同比、市净率、扣非净利润及同比、总股本、每股收益、毛利率、加权 ROE、总市值、每股净资产、资产负债率、每股资本公积、流通 A 股、所属证监会行业、行业市盈率（含更新日期、报表期）。
- **Rox 现状**：诊断有现价、量能、获利盘、PE、市值；可扩展为「主要指标」表格，并增加概念、行业、可比公司（若有数据源）。

### 3. 大事提醒（建议引用）
- **形式**：时间线列表：报表披露、融资融券、投资者互动、最新公告、陆港通、业绩披露、股东人数、分配方案等，带日期与简短说明。
- **Rox 现状**：暂无；若有公告/事件 API，可在诊断下单独一块「大事提醒」。

### 4. 行情分析视图（与专业版分工）
- **头部**：名称、代码、现价、涨跌、涨幅%。
- **图表**：分时/五日；K 线 日/周/月/年/60分/30分/15分/5分；MA5/10/20/60；副图 MACD 等。
- **盘口**：买五卖五、今开/最高/最低/均价、涨停跌停、成交量/额、流通值/总市值、量比/委比、内盘/外盘。
- **财务摘要**：PE(TTM)、PB、PEG、PSM、毛利率、净利率、ROE、负债率。
- **资金流向**：今日资金流向（主力/散户 流入流出 饼图）；5 日资金流向（主买/主卖、超大单/大单/中单/小单净额）；融资余额/融券余量。
- **Rox 现状**：1.0 诊断做「简要行情+结论」；完整 K 线/盘口/资金流向已在 3.0 专业版。可保留「诊断 = 摘要 + 入口到 3.0」，并在诊断页标明「完整行情分析见右侧 ROX 3.0 专业版」。

### 5. 图文 F10 子 Tab（可选）
- **形式**：公司概要、公司大事、主要指标、资讯概览、路演概览、机构调研、互动问答 等 Tab。
- **Rox 现状**：可对应「个股诊断」内左侧分类的细分，或专业版内 Tab 结构。

## 三、资讯与公告（可作新视图或嵌入）

### 1. 新闻研报
- **字段**：日期、类型(新闻/研报)、公司/标的、标题/摘要。
- **用途**：市场看板「综合资讯」或个股诊断「公司资讯/研报评级」。

### 2. 最新公告
- **字段**：日期、类型(股东大会/信息变更/募集资金/董事决议/三季报/股东减持等)、标题/摘要。
- **用途**：市场看板或个股诊断「最新提示/大事提醒」。

### 3. 数据需求
- 需对接新闻、研报、公告 API（或爬虫/第三方），并按市场/个股筛选。

---

## 四、UI/UX 可引用原则

1. **板块独立**：左侧导航 + 主区按「视图」切换，同一屏内不混不同模块内容（已通过 `rox1-view` + `view-active` 落实）。
2. **底部指数条**：任意界面底部可固定一条指数概览，强化「市场温度」。
3. **个股左导航**：个股诊断采用「左侧分类 + 右侧内容」，与通达信「行情」左侧列表一致，便于扩展 F10 类内容。
4. **标题旁入口**：在「个股诊断」标题右侧放「ROX 3.0 专业版」（已实现），与通达信 Tab 旁放功能入口一致。

## 六、建议落地顺序

1. **短期**：市场看板增加「底部指数条」；个股诊断增加「左侧分类导航」框架（先实现 2～3 个分类，其余占位）。
2. **中期**：诊断「公司概要」扩展为主要指标表；板块区增加 Tab 与成分股展开。
3. **长期**：资讯/公告模块（依赖数据源）；大事提醒；多指数分时并排（若需要）。

文档中「建议引用」的项均可直接作为需求写到迭代里；「可选」视资源和数据再排期。
