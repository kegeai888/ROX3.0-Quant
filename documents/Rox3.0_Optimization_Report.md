# Rox3.0 UI 优化与功能增强分析报告

## 1. UI 优化报告 (UI Optimization Report)

### 1.1 现状评估 (Current State)
*   **通达信 (Benchmark)**：
    *   **风格**：高对比度黑底，信息密度极高，几乎无留白。
    *   **交互**：键盘主导（精灵），极速响应。
    *   **视觉**：红绿高亮鲜明，强调数据变动。
*   **Rox3.0**：
    *   **风格**：现代 Web 风格 (Tailwind)，留白较多，适合阅读但不够“硬核”。
    *   **交互**：鼠标主导，缺乏快捷键。
    *   **视觉**：柔和深色模式，缺乏市场跳动的紧迫感。

### 1.2 优化建议 (Recommendations)

#### A. 布局与排版 (Layout & Typography)
*   **紧凑模式 (Compact Mode)**：
    *   将 5 档盘口（Order Book）的行高从 `24px` 压缩至 `18px`。
    *   将自选股列表的 Padding 从 `p-2` 减少为 `p-1` 或 `py-0.5`。
    *   **字体**：数字字段强制使用 `font-mono` (Consolas/Roboto Mono) 并右对齐。

#### B. 交互流程 (Interaction)
*   **键盘精灵 (Smart Command)**：
    *   **核心功能**：监听全局按键。输入 `600` 自动弹出搜索框。
    *   **快捷键**：`Enter` 切换股票，`Up/Down` 选择列表。
    *   **F键系列**：支持 `F1` (分时), `F2` (K线), `F10` (资料)。

#### C. 视觉反馈 (Visual Feedback)
*   **动态闪烁 (Flash Update)**：
    *   价格上涨时背景闪烁深红 (`bg-red-900/30`)，下跌闪烁深绿 (`bg-green-900/30`)，持续 300ms。

## 2. 功能增强建议 (Feature Enhancements)

### 2.1 高价值缺失功能 (High Value Gaps)

| 功能模块 | 通达信实现 | Rox3.0 建议方案 | 价值/必要性 |
| :--- | :--- | :--- | :--- |
| **F10 资料** | 静态文本表格 | **AI 动态研报**：调用 LLM 实时总结财报与新闻。 | **高**：发挥 AI 优势，差异化竞争。 |
| **主力资金** | DDE决策/资金流向 | **资金透视指标**：可视化 `hf_indicator.py` 的计算结果。 | **高**：专业交易者核心关注点。 |
| **自定义公式** | 专有公式语言 | **Python 策略沙箱**：允许用户写 Python 片段定义指标。 | **中**：利用 Python 生态降维打击。 |
| **键盘精灵** | 极其成熟 | **JS 键盘监听器**：复刻该体验。 | **极高**：决定了是否“好用”。 |

### 2.2 实施路线图 (Implementation Roadmap)
1.  **Phase 1 (基础体验)**：实现键盘精灵、紧凑布局、动态价格闪烁。
2.  **Phase 2 (数据可视化)**：接入主力资金数据，在前端绘制资金流向副图。
3.  **Phase 3 (AI 赋能)**：开发“AI F10”面板，点击 F10 触发 AI 分析。

---

## 3. 总结 (Summary)
Rox3.0 不应试图在“行情刷新速度”上与原生 C++ 软件硬碰硬，而应发挥 Web 和 Python 的优势：**更聪明的交互**、**更深度的 AI 分析**、**更现代的扩展性**。
